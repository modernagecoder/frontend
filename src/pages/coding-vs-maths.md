# Coding vs maths: the wrong question, and we love answering it

**The 20-second answer:** neither wins. Coding trains decomposition with instant honest feedback; maths trains abstraction with patience for deep payoff. Which first follows the current need — and the fusion course refuses to choose.

## What each trains

Coding: decomposition, instant feedback (code runs or it does not), expressive creativity, debugging resilience, indirect school payoff. Maths: abstraction, productive stuckness, structural elegance, direct grade-gate payoff. The columns are complementary halves of one education.

## The which-first decision table (from real calls)

Struggling at school maths → maths first (pain compounds monthly, gates doors). Curiosity burning → coding first (motivation engine is free). Ages 10-15 bored by worksheets → the fusion: /courses/maths-through-coding. Strong at both → both, one slot each (most common two-slot pattern). Exam year → maths first, coding after as the reward.

## The overlap, live

The dice argument settler: a child programs 10,000 rolls of two dice to settle whether 7 or 12 is likelier — the coding column gets loops, lists, indexing, a text histogram; the maths column gets sample spaces, frequency vs probability, and 7-has-six-ways read off their own chart. Neither subject is the garnish.
