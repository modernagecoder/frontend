## "CLA C++" — read this first
CLA — C Certified Associate Programmer is for the **C language, not C++**. Many people search "CLA C++" by mistake. The correct **C++** associate certification is **CPA — C++ Certified Associate Programmer**. For C++ professional, it is **CPP**. The C-line is CLA/CLP; the C++ line is CPE/CPA/CPP.

## What CPA is
CPA — C++ Certified Associate Programmer is the associate-level certification from the C++ Institute (a program of OpenEDG). The current exam is CPA-21-02. It validates core C++: the type and operator system, control flow and exceptions, functions and the preprocessor, pointers and dynamic memory, and object-oriented programming with classes, inheritance, polymorphism, encapsulation and namespaces. Path: CPE (entry) → CPA (associate) → CPP (professional).

## Full syllabus (5 blocks)
1. **Types & Operators — 24.5% (9 items):** unary/binary/ternary `?:` operators, precedence & associativity; arithmetic, relational, logical, bitwise, assignment; standard types and ranges; decimal/octal/hex/binary/floating literals; `std::string` (`compare`, `size`, `substr`, `insert`), arrays, vectors, `struct`, `union`, `enum`; conversion, casting, promotion; `signed`/`unsigned`/`static`/`const`.
2. **Control & Exceptions — 18% (8 items):** `if`/`else`; `while`, `do…while`, `for`; `break`, `continue`, `goto`; `switch`/`case`/`default`; `return`; `try`/`catch`/`throw`, catch-all `catch(...)`, exception hierarchies, `throw()` specifier.
3. **Functions & Preprocessor — 17.5% (9 items):** definition/declaration/invocation; typed & `void`; overloading; default parameters; passing by value, reference and pointer; recursion; `main()`; preprocessor `#if`/`#ifdef`/`#else`/`#endif`; parameterized and plain macros.
4. **Pointers — 11% (4 items):** declaration & initialisation; dereferencing and address-of `&`; pointer arithmetic and comparisons; dynamic memory `new`/`delete`/`delete[]` and avoiding leaks.
5. **Classes & Namespaces — 29% (10 items):** OOP (inheritance, encapsulation, polymorphism, abstraction); classes, access specifiers, `::`, `this`; constructors/destructors (default, copy, `explicit`); operator overloading; single & multiple inheritance; `static_cast`/`dynamic_cast`; `virtual` & polymorphic functions; `const` correctness; `friend`; named & anonymous namespaces and aliases.

Classes & Namespaces (29%) and Types & Operators (24.5%) together are over half the exam.

## The C++ Institute certification family
- C++ line: CPE (entry, CPE-20-01) → CPA (associate, CPA-21-02) → CPP (professional, CPP-22-02)
- C line: CLA (associate, CLA-11-03), CLP (professional, CLP-12-01)
- CPA tests core C++ language + OOP; the professional CPP focuses on the Standard Template Library (STL). The C++ Institute recommends CPA before CPP.

## How to prepare
~6–10 weeks at ~6–8 hrs/week with some prior programming (longer from zero). Plan: foundations refresh (weeks 1–2), pointers & memory (week 3), OOP core / Block 5 (weeks 4–6, the heaviest), exceptions & preprocessor (tail), then timed 40-question mocks until consistently above 70%. Code constantly.

## Common pitfalls (FAQ fodder)
Operator precedence and the ternary; pointers vs references; pairing `new`/`delete` to avoid leaks; the object lifecycle and copy constructors; constructors default vs copy vs `explicit`; operator overloading (member vs friend); inheritance access levels; virtual functions and virtual destructors; `static_cast` vs `dynamic_cast`; `const` correctness; function overloading and default arguments; `catch` ordering and `catch(...)`.
