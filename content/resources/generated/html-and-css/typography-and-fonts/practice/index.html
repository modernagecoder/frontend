Topic-Specific Questions

Question 1
Easy
What does this declare?
font-family: 'Poppins', Arial, sans-serif;
Commas mean fallback fonts.
Try Poppins first. If not available, use Arial. If Arial is also missing, use the default sans-serif font.
Question 2
Easy
Why must 'Times New Roman' be in quotes?
The name has spaces.
Because it contains spaces. Font names with spaces must be quoted so the parser treats them as a single name.
Question 3
Easy
What is the default font-weight for normal?
Weights run 100 to 900 in steps of 100.
400
Question 4
Easy
What is the unit-free line-height of 1.6 relative to?
Unitless line-heights multiply the element's font-size.
The element's own font-size. A 20px font with line-height 1.6 gets a 32px line-height.
Question 5
Easy
What does text-transform: uppercase do?
Does it change the HTML?
Displays the text in all capital letters, without changing the actual HTML.
Question 6
Easy
What does font-size: 2rem produce if the root font-size is 16px?
rem = root em.
32px.
Question 7
Easy
What does text-decoration: none do?
Think about what links look like by default.
Removes the underline from links (or the line from tags).
Question 8
Easy
What three values does text-shadow need at minimum?
Think position and color.
x-offset, y-offset, and color. Blur is optional.
Question 9
Easy
What does letter-spacing: 0.1em do?
Think about space between characters.
Adds space equal to 10% of the font size between every character.
Question 10
Easy
Which is a monospace font?
Think code editors.
Fira Code, Courier New, and Monaco are monospace fonts. Every character has the same width.
Question 11
Medium
What is the result?
font-size: clamp(16px, 3vw, 24px);
clamp takes min, preferred, max.
The font-size is at least 16px, grows to 3% of the viewport width at medium sizes, and never exceeds 24px.
Question 12
Medium
What is the font-family of system-ui?
It is a generic family like sans-serif.
Whatever font the user's operating system uses for its UI (San Francisco on macOS, Segoe UI on Windows, Roboto on Android).
Question 13
Medium
Why is unitless line-height better than px for a parent element?
Think about a h1 with bigger font inside.
Unitless line-heights multiply the child's own font-size, so headings and paragraphs each get proportional spacing. Fixed px line-heights inherit as-is and may be too tight for large children.
Question 14
Medium
What does this shorthand set?
font: italic bold 18px/1.5 'Poppins', sans-serif;
Order: style weight size/line-height family.
font-style: italic, font-weight: bold, font-size: 18px, line-height: 1.5, font-family: 'Poppins', sans-serif.
Question 15
Medium
What does &display=swap do in a Google Fonts URL?
Think about what happens while the font is loading.
Tells the browser to show the fallback font immediately, then swap to the web font once it finishes downloading.
Question 16
Medium
What is the difference between em and rem?
What are they each relative to?
em is relative to the parent element's font-size (compounds when nested). rem is relative to the root (html) font-size (no compounding).
Question 17
Medium
What happens if the browser cannot load a Google Font?
Fallbacks.
The browser uses the next font in the font-family fallback list (Arial, Georgia, sans-serif, etc.).
Question 18
Medium
What does text-decoration: underline wavy #a855f7 do?
Modern shorthand allows style and color.
Applies a wavy purple underline under the text.
Question 19
Medium
Which text-align value justifies text to both left and right margins?
Newspaper layout.
justify
Question 20
Medium
What does font-weight: 800 mean?
Weight scale is 100-900.
Extra bold (heavier than regular bold which is 700).
Question 21
Hard
What is the computed line-height in px?
body { font-size: 18px; line-height: 1.6; }
Multiply.
28.8px (18 x 1.6).
Question 22
Hard
Why does this cause a problem?
@font-face { font-family: 'MyFont'; src: url('font.woff2') format('woff2'); }
/* No font-display */
Think about what the user sees while the font is downloading.
The browser hides text until the font loads (FOIT - Flash of Invisible Text), causing blank content for up to 3 seconds.
Question 23
Hard
What is this element's final font-size?
html { font-size: 18px; }
body { font-size: 1rem; }
h1 { font-size: 2em; }
body is 1rem (relative to root), h1 is 2em (relative to body).
36px. (root=18, body=18, h1=2*18=36).
Question 24
Hard
When should you pick a system font stack over Google Fonts?
Think load time and design intent.
Use system fonts when performance is critical (no download), when you want a native platform look, or when the design doesn't need a custom personality. Use Google Fonts when you want a specific brand look or a font that is consistent across all devices.
Question 25
Hard
Why use max-width: 60ch or max-width: 65ch for paragraph text?
ch = width of the '0' character. Think readability studies.
Typography research shows paragraphs are easiest to read when lines contain 45-75 characters. max-width: 65ch limits line length to roughly 65 characters, improving reading comfort and reducing eye fatigue.

Typographers recommend 1.5-1.8.

Question 17
Hard

Shorthand plus clamp plus tighter letter-spacing.
A semibold h1 in Inter with fluid font-size (32-72px), tight line-height (1.1), and slightly tightened letter-spacing for a polished modern heading.
Question 18
Hard
What is the difference between FOIT and FOUT?
Both are font loading issues. One flashes invisible, one flashes unstyled.
FOIT (Flash of Invisible Text) is when the browser hides text completely while the font loads. FOUT (Flash of Unstyled Text) is when the browser shows fallback text first then swaps to the web font. font-display: swap causes FOUT, which is usually preferred.
Question 19
Hard
How does ch unit work?
It is based on a character.
The ch unit equals the width of the '0' (zero) character in the current font. max-width: 60ch limits a container to about 60 characters wide, regardless of font-size.
Question 20
Hard
Why add <link rel="preconnect"> before the Google Fonts link?
Performance.
preconnect tells the browser to start establishing a connection to fonts.googleapis.com and fonts.gstatic.com early, before it actually needs the font. This shaves milliseconds off font loading time.

Multiple Choice Questions

MCQ 1
Which CSS property sets the font?
  • A. font-name
  • B. font-face
  • C. font-family
  • D. text-font
Answer: C
C is correct. font-family sets the typeface. It accepts a list of fallback fonts ending with a generic family.
MCQ 2
Which unit is relative to the ROOT font-size?
  • A. em
  • B. rem
  • C. px
  • D. vw
Answer: B
B is correct. rem = 'root em'. Relative to the html element's font-size. em is relative to the parent.
MCQ 3
What is font-weight: bold numerically?
  • A. 400
  • B. 500
  • C. 600
  • D. 700
Answer: D
D is correct. The keyword bold corresponds to numeric weight 700. normal is 400.
MCQ 4
Which line-height is correct?
  • A. line-height: 16px (best)
  • B. line-height: 1.6 (unitless, best)
  • C. line-height: 160% (wrong)
  • D. line-height: auto
Answer: B
B is correct. Unitless line-heights scale correctly with the element's own font-size. Preferred for almost all cases.
MCQ 5
Which removes the underline from a link?
  • A. text-decoration: remove
  • B. text-decoration: none
  • C. underline: off
  • D. text-style: plain
Answer: B
B is correct. text-decoration: none removes underlines, overlines, and line-throughs.
MCQ 6
How do you display text in all uppercase without changing the HTML?
  • A. font-case: upper
  • B. text-transform: uppercase
  • C. text-style: caps
  • D. font-variant: all-uppercase
Answer: B
B is correct. text-transform: uppercase renders the text in caps visually without modifying the source.
MCQ 7
Which generic font family is for code?
  • A. serif
  • B. sans-serif
  • C. monospace
  • D. cursive
Answer: C
C is correct. monospace guarantees every character has the same width, which is ideal for code.
MCQ 8
How do you center text horizontally?
  • A. align: center
  • B. text-align: center
  • C. text-position: center
  • D. center-text: true
Answer: B
B is correct. text-align: center centers inline content within its block container.
MCQ 9
Which is a web-safe font?
  • A. Poppins
  • B. Inter
  • C. Arial
  • D. Merriweather
Answer: C
C is correct. Arial is preinstalled on virtually every device. The others are web fonts that require loading.
MCQ 10
What does &display=swap do?
  • A. Swaps the font color
  • B. Shows fallback text until the web font loads, then swaps to the web font
  • C. Disables the font
  • D. Rotates through multiple fonts
Answer: B
B is correct. swap keeps text visible during font loading by showing fallback first, then swapping to the web font.
MCQ 11
What does clamp(16px, 2vw, 24px) return?
  • A. Always 16px
  • B. Always 24px
  • C. A value between 16px and 24px based on viewport width
  • D. An error
Answer: C
C is correct. clamp(min, preferred, max) returns the preferred value (2vw), but clamped to never go below 16px or above 24px.
MCQ 12
Which font property quoting is correct?
  • A. font-family: Times New Roman
  • B. font-family: 'Times New Roman', serif
  • C. font-family: "Arial" sans-serif
  • D. font-family: Poppins Bold
Answer: B
B is correct. Names with spaces must be quoted. Multiple fonts are separated by commas, ending with a generic family.
MCQ 13
What is the font shorthand order?
  • A. family size weight
  • B. style weight size/line-height family
  • C. size family weight style
  • D. weight family size
Answer: B
B is correct. The order is: font-style font-variant font-weight font-size/line-height font-family. Family must be last.
MCQ 14
Which is better for accessibility: px or rem for font-size?
  • A. px (it is absolute)
  • B. rem (it respects user's default font-size setting)
  • C. em (it is compact)
  • D. They are identical
Answer: B
B is correct. rem scales with the root font-size, which users can increase for accessibility. px stays fixed regardless.
MCQ 15
Which text-shadow creates a soft glow?

  • B. text-shadow: 0 0 20px red
  • C. text-shadow: 5px 5px red

MCQ 16
What is a reasonable max-width for comfortable reading?
  • A. 1200px
  • B. 65ch or around 680px
  • C. 100%
  • D. 40ch

B is correct. 45-75 characters per line is the sweet spot. 65ch or around 680px work well for body text.
MCQ 17
Why add preconnect links before Google Fonts?
  • A. Required by Google
  • B. Faster font loading by establishing connections early
  • C. Better SEO
  • D. For caching
Answer: B
B is correct. preconnect warms up the DNS and TLS handshake before the browser needs the font file, reducing load time.
MCQ 18
Which declares a custom font from your own file?
  • A. @import-font
  • B. @font-face
  • C. @custom-font
  • D. @load-font
Answer: B
B is correct. @font-face declares a custom font with its file location, format, weight, style, and display strategy.
MCQ 19
What does letter-spacing: -0.02em do to big headings?
  • A. Expands spacing
  • B. Tightens characters slightly, which looks better for large display text
  • C. Has no effect
  • D. Only works for uppercase
Answer: B
B is correct. Big headings look better with slightly tightened letter-spacing. Positive spacing is for small uppercase labels.
MCQ 20
Which is NOT a valid font-weight value?
  • A. 400
  • B. bold
  • C. heavy
  • D. 700
Answer: C
C is correct. 'heavy' is not a CSS keyword. Valid keywords: normal, bold, bolder, lighter. Numeric: 100-900.
MCQ 21
What is the FOIT problem with web fonts?
  • A. The font file is too big
  • B. Text is invisible while the web font downloads
  • C. Fonts look different on mobile
  • D. Fonts cost money
Answer: B
B is correct. Flash of Invisible Text: the browser hides text until the web font loads. font-display: swap fixes this by showing fallback text immediately.
MCQ 22
Why use unitless line-height on the body element?
  • A. It looks nicer
  • B. Unitless values multiply the element's own font-size, so headings and paragraphs get proportional spacing
  • C. It is the only way it works
  • D. It is faster to parse
Answer: B
B is correct. Unitless line-heights scale per-element. A fixed px line-height on body would be inherited as-is and cause overlapping lines on big headings.
MCQ 23
Which combination creates clean modern blog typography?
  • A. Comic Sans, size 11, center aligned
  • B. Merriweather serif body, Inter sans-serif headings, line-height 1.7, max-width 65ch
  • C. Arial bold everywhere, justify align, single line
  • D. Times New Roman 12px, black background
Answer: B
B is correct. A serif for body improves reading comfort. Sans-serif for headings adds contrast. Airy line-height and a sensible max-width round out the recipe.
MCQ 24
Which is the best way to include Google Fonts for performance?
  • A. Load all 9 weights you might ever need
  • B. Load only the 2-3 weights you actually use with display=swap and preconnect
  • C. Download the font files and host them yourself always
  • D. Use @import in the CSS file
Answer: B
B is correct. Minimum weights, swap for visible fallback, preconnect to warm up the connection. @import blocks rendering and is slower than a link tag.
MCQ 25
What is the difference between em and rem when font-size changes?
  • A. They behave identically
  • B. em compounds through nested elements; rem always references the root
  • C. em is pixel-based; rem is percent-based
  • D. rem is older than em
Answer: B
B is correct. em is relative to the parent and compounds: 1.5em inside 1.5em = 2.25x root. rem always references the html element.

Challenge 1: Import and Apply Poppins

Easy
Create a page with a heading 'Hello from Priya' and a short paragraph. Import Poppins from Google Fonts with weights 400 and 800. Apply Poppins to the whole body. Heading should be weight 800, paragraph weight 400.
Sample Input
Open the HTML file
Sample Output
A page with a bold Poppins heading and a regular Poppins paragraph.

<html>
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com">

  <style>
    body { font-family: 'Poppins', sans-serif; padding: 40px; color: #1a1a2e; line-height: 1.6; }
    h1 { font-weight: 800; font-size: 48px; margin: 0 0 12px; }

  </style>
</head>
<body>
  <h1>Hello from Priya</h1>

</body>
</html>
