Topic-Specific Questions

Question 1
Easy
What layout is best for the navbar (logo, links, CTA in a row)?
One row, align to the sides.
Flexbox with justify-content: space-between.
Question 2
Easy
Which CSS function gives fluid font sizes without media queries?
Min, preferred, max.
clamp(min, preferred, max)
Question 3
Easy
Which grid line creates a responsive card layout with no media queries?
The magic line.
grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))
Question 4
Easy
How do you build an FAQ accordion without JavaScript?
Native HTML element.
Use <details> and <summary>. The browser handles the open/close state natively.
Question 5
Easy
What makes a sticky navbar stay at the top while scrolling?
CSS position value.
position: sticky; top: 0
Question 6
Easy
How do you make the hero headline have a gradient color?
Background clip to text.
Use background: linear-gradient(...); -webkit-background-clip: text; background-clip: text; -webkit-text-fill-color: transparent;
Question 7
Easy
What is the purpose of backdrop-filter: blur(10px) on a navbar?
Frosted glass.
It blurs whatever is behind the navbar, creating the 'frosted glass' effect common on modern websites.
Question 8
Easy
How do you make buttons lift slightly on hover?
Transform.
.btn:hover { transform: translateY(-2px); box-shadow: 0 12px 30px rgba(0,0,0,0.2); }
Question 9
Easy
Which property makes the 'MOST POPULAR' ribbon appear above the pricing card without extra HTML?
Pseudo-element.
::before with content: 'MOST POPULAR' and absolute positioning.
Question 10
Easy
Why define CSS variables at the top of the stylesheet?
Theming and maintainability.
Variables let you change brand colors, spacing, and radii in one place. Everything else updates automatically. Much better than find-and-replacing hex codes across 20 rules.
Question 11
Medium
What does this do?
.hero { grid-template-columns: 1.1fr 1fr; }
Ratio of columns.
Two columns where the first takes 1.1 shares and the second takes 1 share. The left column is slightly wider than the right — giving the text more space than the visual.
Question 12
Medium
Why use transform: scale(1.05) on the popular pricing card?
Visual hierarchy.
It makes the card visually pop up from the row so the user's eye is drawn to the recommended plan. Combined with a gradient background and bigger shadow, it is unmissable.
Question 13
Medium
What does this do in the hero?
h1 { text-wrap: balance; }
Line wrapping.
The browser distributes the headline words across lines to avoid a lonely orphan word on the last line. Beautiful typography in one property.
Question 14
Medium
How would you make the pricing section full-width but with centered content?
Section background, inner max-width.
The outer .pricing section has the background color and full width. Inside, a wrapper or the grid itself uses max-width: 1100px; margin: 0 auto.
Question 15
Medium
What does scroll-behavior: smooth on html do?
Nav links with # fragments.
When the user clicks an anchor link like #pricing, the browser smoothly scrolls to that section instead of jumping instantly.
Question 16
Medium
How do you hide the default triangle on <summary>?
List-style and webkit marker.
summary { list-style: none; } summary::-webkit-details-marker { display: none; }
Question 17
Medium
Why is the hero using grid instead of flexbox?
Two-dimensional alignment.
Grid gives precise control over both the column widths (1.1fr 1fr) and vertical alignment (align-items: center). Flexbox would also work for a simple side-by-side, but grid is cleaner for 2D layouts with mixed content sizes.
Question 18
Medium
What CSS makes the feature icons each a different gradient?
nth-child.
Use .feature:nth-child(1) .icon { background: gradient1; }, .feature:nth-child(2) .icon { background: gradient2; }, etc.
Question 19
Medium
What does backdrop-filter: blur(10px) NOT work without?
What has to be transparent?
It does not work unless the element has a partially transparent background. If the element is fully opaque, there is nothing to see through to blur.
Question 20
Medium
Why is the avatar a div with initials instead of an img?
No image assets needed.
A CSS gradient + initials requires zero image files, zero network requests, and zero broken image placeholders. It also scales perfectly at any size.
Question 21
Medium
How does the pricing section change for mobile?
Auto-fit.
Thanks to repeat(auto-fit, minmax(270px, 1fr)), the 3 plans wrap into 1 column on narrow screens. The popular plan also has its scale removed on mobile via a media query.
Question 22
Medium
What does this footer grid do?
.footer-grid { grid-template-columns: 2fr 1fr 1fr 1fr; }
Brand column is wider.
Four columns, where the first is twice as wide as each of the other three. Used so the brand/description column gets more room than the three link columns.
Question 23
Medium
Why use :root for brand colors instead of .brand class?
Global scope.
:root gives the variables global scope so every element in the document can use them. A class like .brand would only make the variable available inside elements with that class.
Question 24
Medium
What is the benefit of using padding: clamp() instead of padding: 32px 64px?
Fluid.
Fixed padding is the same on a phone and a desktop, which wastes space on mobile and looks cramped on desktop. clamp() scales padding with the viewport, so it looks right at every size.
Question 25
Hard
Why use align-items: center on the pricing grid?
Different card heights.
Because the popular plan has an extra list item and is taller. align-items: center centers all cards vertically so the scaled popular card does not look like it is sticking to the top or bottom.
Question 26
Hard
Why are the nav links hidden on mobile?
@media (max-width: 600px) { .nav .links { display: none; } }
Space.
On a 400px phone, the logo, 4 nav links, and a CTA button all crammed in a row would wrap or overflow. Hiding the middle links leaves just the logo and CTA, and we would normally add a hamburger menu in JavaScript.
Question 27
Hard
How would you change the entire page's color theme?
One place.
Edit the three brand variables at the top: --brand, --brand-2, and maybe --accent. Every button, hero gradient, feature icon, popular pricing card, and hover state updates automatically.
Question 28
Hard
What does this do?
html { scroll-padding-top: 80px; }
Sticky navbar plus anchor links.
When a user clicks an anchor link like #pricing, the browser scrolls so that the target is 80px from the top of the viewport, leaving room for the sticky navbar so the heading is not hidden under it.
Question 29
Hard
Why does the hero visual use a nested 2x2 grid of cards?
Depicting a dashboard.
It visually represents a 'dashboard with 4 stats' without needing an actual image. Nested grids are a common technique for building mock product visuals purely in CSS.
Question 30
Hard
Why use :hover effects on pricing cards?
Perceived quality.
Hover interactions make the page feel alive and polished. Lifting the card slightly with translateY(-6px) tells the user 'this is clickable' and adds that premium product feel.

Multiple Choice Questions

MCQ 1
Which layout should you use for the navbar?
  • A. Grid
  • B. Flexbox with space-between
  • C. Float
  • D. Table
Answer: B
B is correct. Flexbox with justify-content: space-between is the simplest and cleanest way to build a navbar with logo on the left, links in the middle, and CTA on the right.
MCQ 2
What CSS line creates a responsive feature grid with no media queries?
  • A. grid-template-columns: repeat(3, 1fr)
  • B. grid-template-columns: repeat(auto-fit, minmax(260px, 1fr))
  • C. grid-template-columns: auto auto auto
  • D. display: flex
Answer: B
B is correct. auto-fit + minmax creates as many columns as fit, wrapping naturally from 1 to 4 columns as the screen size changes.
MCQ 3
Which property makes a navbar stay at the top on scroll?
  • A. position: static
  • B. position: relative
  • C. position: sticky; top: 0
  • D. display: fixed
Answer: C
C is correct. position: sticky with top: 0 keeps the element in place as the user scrolls past its original position.
MCQ 4
How do you build a FAQ accordion without JavaScript?
  • A. Use position: absolute
  • B. Use &lt;details&gt; and &lt;summary&gt;
  • C. Use display: none
  • D. Use visibility: hidden
Answer: B
B is correct. <details> is a native HTML element that handles open/close state without any JavaScript. Style it with CSS for custom visuals.
MCQ 5
What function gives fluid font sizes?
  • A. scale()
  • B. fit-content()
  • C. clamp(min, preferred, max)
  • D. calc()
Answer: C
C is correct. clamp() returns the preferred value clamped between the min and max, giving smooth fluid scaling across screen sizes.
MCQ 6
How do you make a hero headline have gradient text?
  • A. color: linear-gradient(...)
  • B. background: linear-gradient; background-clip: text; color: transparent
  • C. text-color: gradient
  • D. text-shadow: gradient
Answer: B
B is correct. Apply the gradient as a background, clip it to the text with background-clip: text, and make the text transparent so the gradient shows through.
MCQ 7
What is a good max-width for the main content of a landing page?
  • A. 100%
  • B. Around 1100-1200px with margin: 0 auto to center
  • C. 600px always
  • D. No limit
Answer: B
B is correct. 1100-1200px is a sweet spot for readable line lengths and good visual rhythm on wide screens. Use max-width (not fixed width) so it shrinks gracefully.
MCQ 8
Which reset is essential at the start of every stylesheet?
  • A. * { box-sizing: border-box; margin: 0; padding: 0; }
  • B. body { background: white; }
  • C. html { font-size: 16px; }
  • D. div { display: block; }
Answer: A
A is correct. This resets browser default margins and padding and makes box-sizing predictable across every element. Essential for every landing page.
MCQ 9
How should the popular pricing card stand out visually?
  • A. Bigger padding
  • B. transform: scale(1.05), gradient background, larger shadow, and a 'MOST POPULAR' ribbon via ::before
  • C. Different font only
  • D. Make it red
Answer: B
B is correct. A combination of scale, background, shadow, and a ribbon makes the card unmissable. Multiple cues work better than any single one.
MCQ 10
What is the purpose of html { scroll-behavior: smooth }?
  • A. Enables infinite scroll
  • B. Animates anchor link scrolling
  • C. Hides scrollbars
  • D. Disables scrolling
Answer: B
B is correct. When a user clicks a link like #pricing, the browser animates the scroll to the target section instead of jumping instantly.
MCQ 11
Why define CSS variables on :root?
  • A. They are only available inside :root
  • B. They become global and every element can use them
  • C. It is required syntax
  • D. For performance
Answer: B
B is correct. Variables on :root are global and cascade to every element. Changing one value updates the entire page.
MCQ 12
What does backdrop-filter: blur(10px) need to work?
  • A. A solid background
  • B. A semi-transparent background so content behind is visible
  • C. An image
  • D. JavaScript
Answer: B
B is correct. backdrop-filter blurs what is behind the element. A fully opaque background blocks the blur; a semi-transparent background (like rgba(255,255,255,0.9)) lets the frosted glass effect show through.
MCQ 13
Which CSS selects every feature card's icon individually with different gradients?
  • A. .feature:hover .icon
  • B. .feature .icon:first
  • C. .feature:nth-child(n) .icon with rules for n=1,2,3...
  • D. .feature[data-index]
Answer: C
C is correct. :nth-child(n) targets cards by their position so you can assign different gradients without adding custom classes to each.
MCQ 14
Why is max-width preferred over width for containers?
  • A. Width is deprecated
  • B. max-width lets the element shrink on narrow screens without overflowing
  • C. width is not responsive at all
  • D. max-width is faster
Answer: B
B is correct. max-width caps the element at a maximum but lets it shrink naturally on narrow screens. width: 1200px would force a horizontal scrollbar on a phone.
MCQ 15
How do you center a section that has max-width?
  • A. text-align: center
  • B. margin: 0 auto
  • C. display: center
  • D. center: true
Answer: B
B is correct. margin: 0 auto centers a block element horizontally inside its parent when it has a max-width smaller than the parent.
MCQ 16
What is the problem with .hero { width: 1200px; padding: 64px 200px; }?
  • A. Nothing, it is fine
  • B. It overflows narrow screens and creates horizontal scroll
  • C. It is too small
  • D. It uses the wrong units
Answer: B
B is correct. Fixed width 1200px plus padding on narrow screens means the hero is wider than the viewport and creates horizontal scroll. Use max-width and clamp() for padding.
MCQ 17
Why use scroll-padding-top: 80px on html?
  • A. To add scrollbars
  • B. So anchor-link jumps leave room for the sticky navbar, keeping the heading visible
  • C. To fix scroll-snap bugs
  • D. It is required for scroll-behavior
Answer: B
B is correct. Without scroll-padding-top, clicking an anchor link hides the top of the section under the sticky navbar. Padding equal to the navbar height fixes it.
MCQ 18
Why might the popular pricing card use transform: scale(1.05) and reset it on mobile?
  • A. Because scale breaks layouts
  • B. Because scaled cards can overflow narrow mobile screens, so a media query resets transform to none
  • C. Because mobile does not support transforms
  • D. Because the CSS is buggy
Answer: B
B is correct. On narrow screens, the scaled card can touch the edges or overflow. A media query removes the scale on mobile so all cards align within the viewport.
