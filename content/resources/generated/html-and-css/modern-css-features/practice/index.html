Topic-Specific Questions

Question 1
Easy
What does aspect-ratio: 16/9 do?
Think width to height.
Locks the element's width-to-height ratio at 16:9. If you set width to 320px, height becomes 180px automatically.
Question 2
Easy
What does clamp(16px, 5vw, 32px) do?
Minimum, preferred, maximum.
Returns 5vw of the viewport width, but never less than 16px and never more than 32px. A smooth fluid value.
Question 3
Easy
What does :has() select?
It is a parent selector.
An element that contains a descendant matching the inner selector. .card:has(img) matches a card that contains an image.
Question 4
Easy
Why is 100dvh better than 100vh on mobile?
Mobile toolbars change.
On mobile, the address bar shows and hides as the user scrolls. 100dvh (dynamic viewport height) updates to reflect this. 100vh does not, causing layout jumps.
Question 5
Easy
What does text-wrap: balance do on a heading?
No orphans.
It rewraps the heading so each line has roughly the same number of words, avoiding a lonely orphan word on the last line.
Question 6
Easy
What is container-type: inline-size used for?
Marks an element as something.
It marks the element as a query container so descendants can use @container queries based on its width.
Question 7
Easy
What does :focus-visible match?
Keyboard vs mouse.
Focused elements, but only when the browser thinks the focus came from the keyboard (not from a mouse click).
Question 8
Easy
What does margin-inline: 20px do?
Logical property.
Sets margin-left and margin-right each to 20px (in left-to-right languages). In right-to-left languages, it sets right and left accordingly.
Question 9
Easy
What does accent-color: #7c3aed style?
Form controls.
It sets the 'accent' color used to draw checkboxes, radio buttons, range sliders, and progress bars. All native form controls become purple.
Question 10
Easy
Why use container queries instead of media queries?
Components do not know about viewports.
A component does not know how wide the viewport is — it knows how wide its container is. Container queries let the same component adapt correctly whether it is in a narrow sidebar or a wide hero.
Question 11
Medium
What does this do?
.card {
  display: grid;
  grid-template-rows: subgrid;
  grid-row: span 3;
}
Subgrid uses parent tracks.
The card becomes a grid whose rows are inherited from its parent grid. grid-row: span 3 makes it occupy 3 parent rows, and its children snap to those parent tracks.
Question 12
Medium
What does color-mix(in oklab, #7c3aed 80%, white) produce?
Mix 80% purple with 20% white.
A light tint of purple (80% brand purple + 20% white). It is a pastel version of the original color.
Question 13
Medium
What does this do?
body:has(.modal[open]) {
  overflow: hidden;
}
:has() plus attribute.
When any descendant of the body is a modal with the open attribute, the body's scrolling is disabled. Great for locking the background when a modal is visible.
Question 14
Medium
What is the difference between svh, lvh, and dvh?
Small, large, dynamic.
svh: small viewport (with toolbars showing). lvh: large viewport (with toolbars hidden). dvh: dynamic - updates live as toolbars show/hide.
Question 15
Medium
What does this do?
.carousel {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
}
.slide { scroll-snap-align: center; }
Snap to slides.
A horizontal scrolling carousel where each slide snaps to the center of the viewport as the user scrolls. No JavaScript.
Question 16
Medium
What does padding-block-start: 16px do in a left-to-right language?
Block is vertical in LTR.
It sets padding-top: 16px. In LTR languages, the 'block' axis is vertical and 'start' is the top.
Question 17
Medium
Does this container query work?
.sidebar { /* no container-type */ }
@container (min-width: 400px) {
  .card { display: grid; }
}
container-type is required.
No. Without container-type: inline-size (or size) on .sidebar, the browser does not track its size and the @container rule never matches.
Question 18
Medium
What does font-size: clamp(1rem, 3vw + 0.5rem, 2rem) do?
Complex preferred expression.
Font size is 3vw plus 0.5rem, but clamped between 1rem and 2rem. This is a common pattern for smooth but bounded fluid typography.
Question 19
Medium
What selects labels whose input is checked?
Parent selector.
label:has(input:checked)
Question 20
Medium
What is wrong with this?
h1 { font-size: clamp(5vw, 32px, 60px); }
Argument order.
The arguments are in the wrong order. It should be clamp(min, preferred, max), so: clamp(32px, 5vw, 60px).
Question 21
Medium
What does this do?
.thumbnail { aspect-ratio: 1; width: 80px; }
Square.
Creates an 80x80 square. The width is 80px and aspect-ratio: 1 makes the height match.
Question 22
Medium
Is this valid?
input[type="range"] { accent-color: #7c3aed; }
Yes.
Yes. accent-color styles the track and thumb of range sliders with the specified color.
Question 23
Medium
Why is subgrid so useful for card grids?
Alignment across cards.
Without subgrid, each card has its own grid and the internal rows of different cards do not align. With subgrid, cards share the parent's row tracks, so all titles, bodies, and footers line up across the whole row.
Question 24
Medium
Name three things :has() replaces that used to require JavaScript.
Think reactive patterns.
1) Toggling a class on a parent based on child state. 2) Disabling a submit button while any input is invalid. 3) Hiding body scroll when a modal is open. 4) Highlighting a card that contains an image. And many more.
Question 25
Hard
What does this do?
@container card (min-width: 500px) {
  .title { font-size: 28px; }
}
Named container query.
When the nearest ancestor with container-name: card (and a container-type) is at least 500px wide, .title gets a 28px font size.
Question 26
Hard
What is wrong?
body:has(form:has(input:invalid)) { background: red; }
Unnecessary nesting.
Nested :has() is usually unnecessary. Use body:has(input:invalid) instead. :has() already walks the whole subtree.
Question 27
Hard
What happens here?
:root { accent-color: magenta; }
.dark { accent-color: cyan; }
Inheritance.
Form controls use magenta by default but cyan inside any element with the dark class. accent-color is inherited.
Question 28
Hard
Does this do what you expect?
.card:has(.featured) h3 {
  color: red;
}
Parent selector with a descendant combinator.
Yes. It selects h3 elements inside a card that contains a .featured descendant. The red color applies only to cards marked as featured.
Question 29
Hard
What does interpolate-size: allow-keywords do?
Animating to/from auto.
It lets CSS transition from a fixed size to auto (or min-content, max-content, etc.), which historically was impossible. Enables animating accordions open without JavaScript.
Question 30
Hard
When would you use container-type: size instead of inline-size?
Both axes.
Use size when you need to query the container's height as well. It tracks both width and height. Use inline-size (more efficient) when you only need width, which is most of the time.
Question 31
Hard
Why is color-mix(in oklab, ...) preferred over in srgb?
Color space.
oklab is a perceptually-uniform color space, so mixing produces visually smooth transitions without the muddy greens and browns you sometimes get in sRGB. Mixing red and blue in oklab gives a clean purple.
Question 32
Hard
How does text-wrap: balance work differently from manual <br> tags?
Responsive.
Manual line breaks force a break at a fixed position regardless of screen size, often breaking badly on narrow or wide screens. text-wrap: balance lets the browser rebalance at any size, so the heading always looks well-wrapped.

Multiple Choice Questions

MCQ 1
What does aspect-ratio: 16/9 do?
  • A. Sets the width to 16px and height to 9px
  • B. Locks the ratio of width to height at 16:9
  • C. Crops the element to 16x9
  • D. Resizes the image file
Answer: B
B is correct. aspect-ratio locks the width-to-height ratio. Set one dimension and the other is calculated automatically.
MCQ 2
Which is the correct order for clamp()?
  • A. clamp(preferred, min, max)
  • B. clamp(max, preferred, min)
  • C. clamp(min, preferred, max)
  • D. clamp(min, max, preferred)
Answer: C
C is correct. clamp(min, preferred, max). The preferred value is used, but never below the min or above the max.
MCQ 3
What does :has() enable that CSS historically could not do?
  • A. Style child elements
  • B. Select a parent based on its children
  • C. Add animations
  • D. Change fonts
Answer: B
B is correct. :has() is the parent selector. It lets you match an element based on what it contains, which CSS could not do before 2022-2023.
MCQ 4
What must an element have to participate in container queries?
  • A. display: container
  • B. container-type: inline-size (or size)
  • C. class="container"
  • D. Nothing, it works by default
Answer: B
B is correct. Without container-type, the browser does not track the element's size for container queries.
MCQ 5
Which unit should you prefer over 100vh on mobile?
  • A. 100vw
  • B. 100dvh
  • C. 100em
  • D. 100%
Answer: B
B is correct. 100dvh (dynamic viewport height) adjusts as mobile browser toolbars show and hide, preventing layout jumps.
MCQ 6
What does text-wrap: balance do?
  • A. Centers text
  • B. Distributes words across lines to avoid orphans
  • C. Makes text bold
  • D. Changes font size
Answer: B
B is correct. It rebalances wrapping so a heading does not end with a single orphan word. Great for hero titles.
MCQ 7
Which property themes all native form controls in one line?
  • A. form-color
  • B. control-color
  • C. accent-color
  • D. theme-color
Answer: C
C is correct. accent-color sets the theme color for checkboxes, radios, range sliders, and progress bars.
MCQ 8
What is margin-inline: 20px in a left-to-right language?
  • A. 20px top and bottom
  • B. 20px left and right
  • C. 20px only left
  • D. Not valid CSS
Answer: B
B is correct. In LTR languages, the inline axis is horizontal. margin-inline sets left and right margin. In RTL languages it still sets the start-and-end direction correctly.
MCQ 9
What does container-type: inline-size mean?
  • A. Only track the width
  • B. Only track the height
  • C. Track both width and height
  • D. Disable queries
Answer: A
A is correct. inline-size tracks the container's inline (width) dimension. It is more efficient than size and is what you want most of the time.
MCQ 10
What is subgrid used for?
  • A. Adding rows to a grid
  • B. Making a nested grid share its parent's tracks
  • C. Hiding the grid
  • D. Changing column count
Answer: B
B is correct. grid-template-columns: subgrid (or rows) makes a child grid use the parent's tracks, aligning descendants across siblings.
MCQ 11
Which :focus-visible statement is TRUE?
  • A. It only matches on mouse clicks
  • B. It matches when the browser decides the focus is keyboard-like
  • C. It is the same as :focus
  • D. It only works on inputs
Answer: B
B is correct. :focus-visible is the modern focus ring pseudo-class. It shows the ring for keyboard users but hides it for mouse users who do not need it.
MCQ 12
How do you disable a submit button while any input is invalid, with just CSS?
  • A. Impossible without JavaScript
  • B. form:has(input:invalid) button { pointer-events: none; opacity: 0.4; }
  • C. input:invalid button { disabled: true; }
  • D. form[invalid] button { display: none; }
Answer: B
B is correct. :has() checks if the form contains an invalid input. If so, the rule disables and fades the button.
MCQ 13
What does color-mix(in oklab, red 50%, blue) produce?
  • A. A 50/50 blend of red and blue in oklab color space (a clean purple)
  • B. Only red
  • C. Only blue
  • D. A random color
Answer: A
A is correct. color-mix() blends two colors in the specified color space. oklab gives perceptually-uniform mixing.
MCQ 14
Which is the right way to build a carousel with pure CSS?
  • A. Use JavaScript
  • B. Use scroll-snap-type on the container and scroll-snap-align on each slide
  • C. Use display: table
  • D. Use position: fixed
Answer: B
B is correct. scroll-snap gives you native carousel scrolling that snaps to each slide, with no JavaScript required.
MCQ 15
Which viewport unit updates live as mobile toolbars show/hide?
  • A. vh
  • B. svh
  • C. lvh
  • D. dvh
Answer: D
D is correct. dvh (dynamic viewport height) updates continuously to match the current visible viewport.
MCQ 16
You have @container (min-width: 400px) { ... } with NO container-type set on any ancestor. What happens?
  • A. It falls back to media query
  • B. It never matches — nothing has been declared as a container
  • C. It matches the viewport
  • D. It throws an error
Answer: B
B is correct. Without container-type on any ancestor, the query has no container to match and never activates.
MCQ 17
Which is the best fluid font-size declaration?
  • A. font-size: 5vw
  • B. font-size: 24px
  • C. font-size: clamp(16px, 2.5vw + 0.5rem, 32px)
  • D. font-size: 2vh
Answer: C
C is correct. clamp() gives a fluid value bounded by a readable minimum and a reasonable maximum. Pure vw becomes too small on mobile and too large on desktop.
MCQ 18
What does this do?
label:has(input:checked) { background: #7c3aed; color: white; }
  • A. Styles all labels
  • B. Styles only labels that contain a currently-checked input
  • C. Styles the input
  • D. Nothing
Answer: B
B is correct. :has(input:checked) matches labels with a checked input inside. Clicking the label ticks the input, which instantly triggers the highlight style.

Challenge 1: Fluid Typography Hero

Easy
Build a full-screen hero section with a large heading, subtitle, and CTA button. Use clamp() for every font-size and padding value so the whole section scales smoothly from mobile to desktop. Use text-wrap: balance on the heading. Use 100dvh for the height. Name the project after Aarav.
Sample Input
Resize the browser window
Sample Output
A hero that scales smoothly from phone to desktop without any media queries.
No media queries. Use clamp() for fluid values. Use 100dvh. Use text-wrap: balance.

<html>
<head>

    min-height: 100dvh;

Challenge 3: Card That Responds to Its Container

Medium
Build a card component that uses container queries. Place the same card in two containers: a narrow column and a wide column. When the container is wider than 380px, the card should switch from vertical (image above text) to horizontal (image beside text). Use container-type: inline-size and @container.
Sample Input
None
Sample Output
Two copies of the same card component with different layouts based on their container widths.
Use container queries. No media queries. Same CSS class for both cards.
<!DOCTYPE html>
<html>
<head>
<style>
  * { box-sizing: border-box; margin: 0; padding: 0; }
  body { font-family: 'Segoe UI', sans-serif; background: #f1f5f9; padding: 32px; display: grid; gap: 24px; grid-template-columns: 1fr 2fr; max-width: 1100px; margin: 0 auto; }
  @media (max-width: 800px) { body { grid-template-columns: 1fr; } }
  .column { background: white; padding: 20px; border-radius: 14px; container-type: inline-size; box-shadow: 0 4px 14px rgba(0,0,0,0.06); }
  .column h2 { color: #7c3aed; margin-bottom: 14px; font-size: 16px; text-transform: uppercase; letter-spacing: 1px; }
  .card {
    background: #faf5ff;
    border: 1px solid #e9d5ff;
    border-radius: 12px;
    padding: 16px;
  }
  .card .thumb {
    width: 100%;
    aspect-ratio: 16/10;
    background: linear-gradient(135deg, #a855f7, #ec4899);
    border-radius: 8px;
    margin-bottom: 12px;
  }
  .card h3 { color: #0f172a; font-size: 18px; margin-bottom: 6px; }
  .card p { color: #64748b; font-size: 14px; line-height: 1.5; }
  @container (min-width: 380px) {
    .card {
      display: grid;
      grid-template-columns: 160px 1fr;
      gap: 16px;
      align-items: center;
    }
    .card .thumb { aspect-ratio: 1; margin: 0; }
  }
</style>
</head>
<body>
  <div class="column">
    <h2>Narrow column</h2>
    <div class="card">
      <div class="thumb"></div>
      <h3>Meera's Blog Post</h3>
      <p>This column is narrow, so the same card stacks vertically with the image on top.</p>
    </div>
  </div>
  <div class="column">
    <h2>Wide column</h2>
    <div class="card">
      <div class="thumb"></div>

</html>

Challenge 5: Subgrid Card Row with Aligned Content

Hard
Build a row of 3 blog cards where the title, body, and 'Read More' link align perfectly across all cards, even though the content lengths differ. Use grid-template-rows: subgrid on each card. The parent grid must define the row structure.
Sample Input
None
Sample Output
Three cards where each title starts at the same y-position and each 'Read More' sits at the same y-position, regardless of body length.
Use subgrid. Do not use fixed heights on the cards.
<!DOCTYPE html>
<html>
<head>
<style>
  * { box-sizing: border-box; margin: 0; padding: 0; }
  body { font-family: 'Segoe UI', sans-serif; background: #f1f5f9; padding: 40px 24px; }
  h1 { text-align: center; color: #7c3aed; margin-bottom: 24px; font-size: 32px; }
  .grid {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
    grid-template-rows: auto auto 1fr auto;
    gap: 20px;
    max-width: 1000px;
    margin: 0 auto;
  }
  .card {
    display: grid;
    grid-row: span 4;
    grid-template-rows: subgrid;
    background: white;
    border-radius: 16px;
    padding: 24px;
    box-shadow: 0 8px 24px rgba(0,0,0,0.08);
    transition: transform 0.2s;
  }
  .card:hover { transform: translateY(-4px); }

Type into form fields
Sample Output
The form reacts to validation state in real time: red border + disabled button when invalid, green border + enabled button when all valid.
