Topic-Specific Questions

Question 1
Easy
What does this select?
button:hover { background: purple; }
Think about when.
Buttons when the mouse is pointing at them. The background turns purple during hover and reverts when the mouse leaves.
Question 2
Easy
How many colons before 'before'?
.card___before { content: ''; }
Pseudo-elements got a double colon in CSS3.
Two colons: .card::before.
Question 3
Easy
What does li:first-child select?
Which one in the list?
The first li inside its parent.
Question 4
Easy
What does ::before need to actually render?
A required property.
The content property. Even content: '' (empty string) works.
Question 5
Easy
Which rows does this style?
tr:nth-child(odd) { background: #f1f5f9; }
Odd means 1, 3, 5...
Rows 1, 3, 5, 7... (the 1st, 3rd, 5th, etc.)
Question 6
Easy
What does input:checked select?
For checkboxes and radios.
Checkboxes and radio buttons that are currently ticked.
Question 7
Easy
What does a:visited target?
Browser history.
Links the user has already visited (according to browser history).
Question 8
Easy
Which element does p::first-letter target?
Drop caps.
The very first character of the paragraph.
Question 9
Easy
What is the difference between :focus and :focus-visible?
Mouse vs keyboard.
:focus matches any focused element (including clicks). :focus-visible only matches when the browser determines the focus is keyboard-driven, so mouse users do not see outlines.
Question 10
Easy
Why is content: attr(data-tip) powerful?
It reads from HTML.
It lets a pseudo-element display text from an HTML attribute. Each element can have different tooltip text set via data-tip="...", and one CSS rule handles them all.
Question 11
Medium
Which items does this select?
li:nth-child(3n+2) { color: red; }
Start n at 0: 3(0)+2, 3(1)+2, 3(2)+2...
Items 2, 5, 8, 11...
Question 12
Medium
What does this do?
li:not(.active) { opacity: 0.5; }
Negation.
Makes every li that does NOT have the class active 50% transparent. The active item stays fully visible.
Question 13
Medium
What selector groups these?
h1, h2, h3 { font-family: 'Syne'; }
Use a function that groups.
:is(h1, h2, h3) { font-family: 'Syne'; }
Question 14
Medium
What is the difference between :is() and :where()?
Specificity.
:is() takes the highest specificity of its arguments. :where() has zero specificity, which makes it easy to override.
Question 15
Medium
What does this select?
label:has(input:checked) { background: #7c3aed; }
:has() is the parent selector.
Labels that contain a checked input. The label itself turns purple when the checkbox inside it is ticked.
Question 16
Medium
What renders here?
.new::after {
  content: 'NEW';
  color: red;
}
Injected text.
The text 'NEW' in red appears at the end of every element with class new.
Question 17
Medium
Is this valid? 
input:invalid { border-color: red; }
Form validation pseudo-class.
Yes. :invalid matches form inputs whose value does not satisfy their validation rules (email, required, minlength, min/max, pattern, etc.).
Question 18
Medium
What happens when the user selects text?
::selection { background: #a855f7; color: white; }
Highlighted text styling.
When the user highlights any text on the page, the background becomes purple and the text becomes white.
Question 19
Medium
What does this style?
li::marker { color: #7c3aed; font-size: 1.3em; }
List item decoration.
The bullet point (or number) of each list item becomes purple and slightly larger.
Question 20
Medium
What does div:empty { display: none; } do?
Empty means no content at all.
Any div that has no content (no text, no children, not even whitespace) is hidden.
Question 21
Medium
What does this do?
section:target { background: #fef3c7; }
URL fragments.
When the URL contains #sectionId matching the section's id, that section gets a yellow background.
Question 22
Medium
Which elements does this style?
input:placeholder-shown { font-style: italic; }
When the placeholder is visible.
Inputs that are currently showing their placeholder (i.e., empty inputs with a placeholder attribute).
Question 23
Medium
Why should you pair :hover with :focus-visible?
Keyboard users.
Keyboard users cannot hover. Pairing :hover with :focus-visible ensures they see the same feedback as mouse users when they tab to an element.
Question 24
Medium
What is the purpose of :where() in a CSS library?
Think about overrides.
:where() has zero specificity, so base styles inside it are trivially overridable by the user's own CSS. Libraries use it to provide defaults that do not fight with user code.
Question 25
Hard
What does this select?
article p:first-of-type::first-line { font-weight: bold; }
Combine a pseudo-class and a pseudo-element.
The first line of text in the first p element inside any article. Just that first line is bold.
Question 26
Hard
What does this select?
form:has(input:invalid) button[type="submit"] { opacity: 0.4; pointer-events: none; }
:has() checks descendants.
The submit button of any form that contains an invalid input. The button becomes faded and unclickable until all inputs become valid.
Question 27
Hard
What is odd about this?
.cards > div:nth-child(3n) { border: 2px solid red; }
Direct children only.
Only direct div children of .cards are considered. Nested divs inside other children are ignored. Every third direct child gets a red border.
Question 28
Hard
Does this work as an animated underline?
a::after {
  content: '';
  display: block;
  width: 0;
  height: 2px;
  background: #a855f7;
  transition: width 0.3s;
}
a:hover::after { width: 100%; }
Yes, but check direction.
Yes. On hover, the underline grows from width 0 to 100% over 0.3s. The direction is left-to-right (default origin).
Question 29
Hard
What is wrong with this?
.product.sale::before {
  content: 'SALE';
}
.product::before {
  content: '';
  background: #f0f0f0;
  width: 100%; height: 100%;
}
Only one ::before per element.
An element can only have ONE ::before. The more specific selector wins. You cannot have two ::before pseudo-elements on the same element stacking.
Question 30
Hard
How would you disable a submit button while any form input is invalid, without JavaScript?
Use :has() and :invalid.
form:has(:invalid) button[type="submit"] { opacity: 0.4; pointer-events: none; }
Question 31
Hard
Why is :hover a bad idea as the only interaction trigger on mobile?
Touch devices.
Touch devices do not have a persistent hover state. A finger tap is a click, not a hover. Relying only on :hover to reveal important content makes mobile users miss it entirely.
Question 32
Hard
What problem does ::first-letter solve that extra HTML would also solve?
Semantics.
It styles the first character for a drop cap effect without wrapping it in a <span>. Keeps the HTML clean and semantic — the visual decoration does not pollute the content.

Multiple Choice Questions

MCQ 1
Which selector styles the first child of its parent?
  • A. :first-of
  • B. :first-child
  • C. :child-first
  • D. :only-child
Answer: B
B is correct. :first-child matches an element that is the first child of its parent.
MCQ 2
How many colons does a pseudo-element use in modern CSS?
  • A. Zero
  • B. One
  • C. Two
  • D. Three
Answer: C
C is correct. Pseudo-elements use two colons (::before, ::after, ::first-letter) in CSS3+. Pseudo-classes use one colon.
MCQ 3
What property is required for ::before to render?
  • A. display
  • B. content
  • C. visibility
  • D. position
Answer: B
B is correct. Without a content property, ::before and ::after do not render at all. Even content: '' is enough.
MCQ 4
Which pseudo-class matches when the mouse is over the element?
  • A. :click
  • B. :hover
  • C. :point
  • D. :over
Answer: B
B is correct. :hover matches when the pointer is over the element and is one of the most common interaction styles.
MCQ 5
Which selector picks every even list item?
  • A. li:nth-child(even)
  • B. li:even-child
  • C. li:even
  • D. li::even
Answer: A
A is correct. :nth-child(even) matches every even child (2nd, 4th, 6th...). odd and formulas like 2n+1 also work.
MCQ 6
Which pseudo-class matches a checked checkbox?
  • A. :selected
  • B. :checked
  • C. :ticked
  • D. :active
Answer: B
B is correct. :checked matches checkboxes and radios that are currently in the checked state, as well as selected options.
MCQ 7
How do you style selected text on a page?
  • A. :selected
  • B. ::highlight
  • C. ::selection
  • D. :active
Answer: C
C is correct. ::selection styles text that the user has currently highlighted.
MCQ 8
Which selector targets the placeholder text inside an input?
  • A. input:placeholder
  • B. input::placeholder
  • C. input.placeholder
  • D. input:hint
Answer: B
B is correct. ::placeholder is a pseudo-element (double colon) that styles the placeholder text inside form inputs.
MCQ 9
What does li:nth-child(3n) select?
  • A. The first 3 items
  • B. Items 1, 2, 3
  • C. Every 3rd item (3, 6, 9...)
  • D. Items after the third
Answer: C
C is correct. 3n with n starting at 0 gives 0, 3, 6, 9... The 0th does not exist, so the pattern is 3rd, 6th, 9th, etc.
MCQ 10
How do you read an HTML attribute as content inside ::after?
  • A. content: var(data-tip)
  • B. content: attr(data-tip)
  • C. content: $data-tip
  • D. content: @data-tip
Answer: B
B is correct. attr() reads an HTML attribute value. It is the standard way to drive tooltips and badges from data-* attributes.
MCQ 11
Which selector has zero specificity?
  • A. :is()
  • B. :not()
  • C. :where()
  • D. :has()
Answer: C
C is correct. :where() has zero specificity, making its styles trivially overridable. :is() takes the specificity of its most specific argument.
MCQ 12
What does form:has(:invalid) button select?
  • A. Buttons that are invalid
  • B. Buttons inside a form that contains any invalid field
  • C. Invalid forms only
  • D. Nothing - :has() does not exist
Answer: B
B is correct. :has() is the parent selector: it matches a form that contains an invalid descendant, then the rule applies to buttons inside that form.
MCQ 13
Which pseudo-class only applies when the focus came from the keyboard?
  • A. :focus
  • B. :focus-visible
  • C. :keyboard
  • D. :tab
Answer: B
B is correct. :focus-visible is shown only when the browser determines the focus is keyboard-driven, keeping the UI clean for mouse users while remaining accessible.
MCQ 14
Which selector matches the element whose id is the URL fragment?
  • A. :anchor
  • B. :url
  • C. :target
  • D. :fragment
Answer: C
C is correct. :target matches the element whose id corresponds to the current URL fragment (the part after #).
MCQ 15
What does :not(.active) match?
  • A. Elements with class active
  • B. Elements without the class active
  • C. All elements
  • D. Only the active element
Answer: B
B is correct. :not(selector) matches elements that do NOT match the inner selector. This is a clean way to style everything except one item.
MCQ 16
Can you have two ::before pseudo-elements on the same element?
  • A. Yes, they stack
  • B. No, an element has only one ::before
  • C. Yes, if they have different content
  • D. Only with JavaScript
Answer: B
B is correct. Each element has exactly one ::before and one ::after. Multiple rules for the same pseudo-element merge via normal cascade.
MCQ 17
Which selector picks only the first paragraph inside each article?
  • A. article p:first
  • B. article p:first-child
  • C. article p:first-of-type
  • D. Both B and C can work - C is safer if the article has other elements first
Answer: D
D is correct. :first-child only matches if the p is literally the first child. :first-of-type matches the first p regardless of what other elements come before it. Choose based on your HTML.
MCQ 18
What does input:not(:placeholder-shown):invalid match?
  • A. Empty inputs
  • B. Inputs the user has typed in AND which are currently invalid
  • C. All invalid inputs
  • D. All inputs with placeholders
Answer: B
B is correct. :not(:placeholder-shown) means 'the placeholder is NOT showing', i.e., the input has content. Combined with :invalid, it only shows red errors after the user starts typing.

Challenge 2: Custom Tooltip with ::after

Easy
Create 4 buttons with different hover tooltips. The tooltip text must come from a data-tip attribute using content: attr(data-tip). The tooltip should appear above the button on hover with a small arrow pointing down, and fade in smoothly.
Sample Input
Hover over any button
Sample Output
A dark tooltip with the attribute text appears above the button with a fade-in animation and an arrow.
Use ::after for the tooltip body, ::before for the arrow. Use attr() for content. No JavaScript.
<!DOCTYPE html>
<html>
<head>
<style>
  body { font-family: 'Segoe UI', sans-serif; background: #0f172a; padding: 100px 40px; display: flex; gap: 30px; justify-content: center; flex-wrap: wrap; }
  .btn { position: relative; padding: 14px 24px; background: #7c3aed; color: white; border: none; border-radius: 10px; font-weight: 600; cursor: pointer; font-size: 15px; }
  .btn::after {
    content: attr(data-tip);
    position: absolute;
    left: 50%;
    bottom: calc(100% + 12px);
    transform: translateX(-50%);
    background: #0b1120;
    color: #e2e8f0;
    padding: 8px 14px;
    border-radius: 8px;
    font-size: 13px;
    white-space: nowrap;
    box-shadow: 0 8px 20px rgba(0,0,0,0.4);
    opacity: 0;
    pointer-events: none;
    transition: opacity 0.2s, transform 0.2s;
  }
  .btn::before {
    content: '';
    position: absolute;
    left: 50%;
    bottom: calc(100% + 6px);
    transform: translateX(-50%);
    border: 6px solid transparent;
    border-top-color: #0b1120;
    opacity: 0;
    transition: opacity 0.2s;
  }
  .btn:hover::after, .btn:hover::before {
    opacity: 1;
    transform: translateX(-50%) translateY(-4px);
  }
</style>
</head>
<body>
  <button class="btn" data-tip="Start Aarav's quiz now">Start Quiz</button>
  <button class="btn" data-tip="Save Priya's progress">Save</button>
  <button class="btn" data-tip="Export Rohan's report">Export</button>
  <button class="btn" data-tip="Warning: Delete Ishita's project">Delete</button>
</body>
</html>

Medium

Challenge 6: Pure CSS Accordion with :has() and :target

Hard
Build a FAQ accordion with 4 questions using only HTML and CSS. Use <details> and <summary> elements. Style the open state with [open] or ::marker. Use ::before to add a plus/minus indicator that changes when the section is expanded. No JavaScript.
Sample Input
Click a question to expand it
Sample Output
A clean FAQ with plus icons that turn into minus icons when expanded, and smooth hover states.
Use only HTML and CSS. Use details/summary. Use pseudo-elements for the indicator.
<!DOCTYPE html>
<html>
<head>
<style>
  body { font-family: 'Segoe UI', sans-serif; background: #f8fafc; padding: 40px; max-width: 680px; margin: 0 auto; }
  h1 { color: #7c3aed; text-align: center; margin-bottom: 24px; }
