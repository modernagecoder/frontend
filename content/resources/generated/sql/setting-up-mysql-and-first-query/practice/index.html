Question 20
Hard
What is the difference between MySQL Workbench and the MySQL CLI, and when would you use each?
GUI vs terminal tradeoffs.
Workbench is a GUI with visual query editor, result grid, schema browser, and EER diagram tool — great for exploration and learning. CLI is a terminal client — faster for experienced users, the only option on servers without a desktop, and essential when writing shell scripts or SSH'ing into a remote server.

A junior in your team wrote: SELECT * FROM school.students;. The senior says "never use SELECT *". Rewrite it to return only the name and marks columns, and explain why the senior prefers that.

Multiple Choice Questions

MCQ 1
What is the default port for MySQL?
  • A. 1433
  • B. 3306
  • C. 5432
  • D. 8080
Answer: B
B is correct. MySQL defaults to port 3306. SQL Server uses 1433, PostgreSQL uses 5432, and 8080 is a common HTTP port.
MCQ 2
Which command lists all databases on a MySQL server?
  • A. LIST DATABASES;
  • B. SHOW DATABASES;
  • C. SELECT DATABASES;
  • D. DISPLAY DATABASES;
Answer: B
B is correct. SHOW DATABASES; is the MySQL command. The other options are not valid SQL.
MCQ 3
Which command sets the current active database to school?
  • A. SELECT school;
  • B. OPEN school;
  • C. USE school;
  • D. CONNECT school;
Answer: C
C is correct. USE dbname; selects a database for the session. The others are not valid MySQL commands.
MCQ 4
Which command shows the structure (columns, types) of a table?
  • A. STRUCT students;
  • B. DESCRIBE students;
  • C. LIST students;
  • D. SHOW students;
Answer: B
B is correct. DESCRIBE students; (or its shortcut DESC students;) prints the schema. SHOW students; is not valid — you use SHOW TABLES; to list tables.
MCQ 5
What character terminates an SQL statement in the MySQL CLI?
  • A. A period (.)
  • B. A comma (,)
  • C. A semicolon (;)
  • D. A newline
Answer: C
C is correct. Every SQL statement ends with a semicolon. A newline alone doesn't terminate — MySQL waits for the ; before executing.
MCQ 6
Which of the following is a valid single-line SQL comment?
  • A. // this is a comment
  • B. -- this is a comment
  • C. <!-- this is a comment -->
  • D. ** this is a comment **
Answer: B
B is correct. -- (two dashes and a space) starts a portable SQL comment. // is C/Java comment syntax. <!-- --> is HTML. ** is not a valid comment.
MCQ 7
You connect to MySQL and type mysql> SELECT * FROM students then press Enter without a semicolon. What happens?
  • A. Error is raised
  • B. Query runs successfully
  • C. Prompt changes to -> waiting for the statement to finish
  • D. Client disconnects
Answer: C
C is correct. The CLI displays -> and waits for you to finish with a semicolon. Type ; on the next line to execute.
MCQ 8
Which INSERT statement is valid for a students table with columns (id INT, name VARCHAR(50), marks INT)?
  • A. INSERT INTO students VALUES (1, "Aarav", 87);
  • B. INSERT INTO students VALUES (1, 'Aarav', 87);
  • C. INSERT students VALUES (1, 'Aarav', 87);
  • D. INSERT (1, 'Aarav', 87) INTO students;
Answer: B
B is correct. Standard SQL uses single quotes for strings. INSERT INTO table VALUES (...) is the correct keyword order. Option A uses double quotes (works on MySQL but not portable). Options C and D have wrong syntax.
MCQ 9
You run SELECT DATABASE(); right after logging in (without any USE). What do you see?
  • A. mysql
  • B. information_schema
  • C. NULL
  • D. Error
Answer: C
C is correct. Without a USE statement, no database is active, so DATABASE() returns NULL. It is not an error — you still get a result, just NULL.
MCQ 10
Which statement creates a table ONLY if it does not already exist?
  • A. CREATE TABLE students (...)
  • B. CREATE OR REPLACE TABLE students (...)
  • C. CREATE TABLE IF NOT EXISTS students (...)
  • D. CREATE TABLE students IF NOT EXISTS (...)
Answer: C
C is correct. The IF NOT EXISTS clause goes after TABLE and before the table name. Makes scripts re-runnable. CREATE OR REPLACE TABLE exists in some databases (like MariaDB) but not standard MySQL.
MCQ 11
A senior developer tells you "avoid SELECT *". Which is NOT a reason for this advice?
  • A. Reduces data sent over the network
  • B. Makes the query slower even on small tables
  • C. Result shape stays stable if a new column is added later
  • D. Makes intent more explicit
Answer: B
B is correct (it is NOT a valid reason). On small tables SELECT * is not noticeably slower. The real reasons are A, C, and D. On large tables the network cost matters; on small ones, readability and stability matter more.
MCQ 12
In the CLI, which keyboard sequence ends the session and returns to your OS shell?
  • A. exit;
  • B. quit;
  • C. \q
  • D. All of the above
Answer: D
D is correct. exit;, quit;, and the shortcut \q all disconnect from the MySQL server and return you to your terminal prompt.
MCQ 13
Which statement about VARCHAR(50) is TRUE?
  • A. It always takes 50 bytes per row, even for shorter strings
  • B. It stores up to 50 characters but only uses space for the actual length plus a small overhead
  • C. It can only store exactly 50 characters
  • D. It is limited to ASCII characters
Answer: B
B is correct. VARCHAR is variable-length: "Ram" takes 3 bytes plus 1-2 bytes of length prefix, not 50. CHAR(50) would pad to exactly 50. VARCHAR can store any UTF-8 character if the column's charset supports it.
MCQ 14
A friend writes INSERT INTO students VALUES (1, 'Aarav'); for a students table with 4 columns. What happens?
  • A. Inserts the row with NULL for missing columns
  • B. Inserts successfully ignoring extras
  • C. Error: Column count doesn't match value count
  • D. Inserts 0 for missing columns
Answer: C
C is correct. Without naming columns, MySQL expects values for ALL columns in declared order. To insert just some columns, you must list them: INSERT INTO students (id, name) VALUES (1, 'Aarav');.
MCQ 15
Given a clean school database, what does this sequence produce?
CREATE TABLE t (x INT);
INSERT INTO t VALUES (10);
DROP TABLE t;
CREATE TABLE t (x INT);
SELECT * FROM t;
  • A. One row with x=10
  • B. Empty set (no rows)
  • C. Error: table already exists
  • D. Error: unknown table
Answer: B
B is correct. The DROP deletes both the structure and the data. The second CREATE makes a fresh empty table. SELECT returns Empty set.
MCQ 16
Which command would you use at a shared hosting server where only a CLI is available, to import a large .sql dump file?
  • A. SOURCE dump.sql; from inside the mysql client
  • B. IMPORT dump.sql; from SQL
  • C. Use MySQL Workbench GUI
  • D. LOAD dump.sql INTO DATABASE;
Answer: A
A is correct. SOURCE /path/to/dump.sql; (or its shortcut \. /path/to/dump.sql) runs every statement in the file. Alternatively, from the OS shell: mysql -u root -p dbname < dump.sql. There is no IMPORT or LOAD dump.sql command.
MCQ 17
You type INSERT INTO students VALUES (1, "Aarav", 10, 87); on a MySQL server with ANSI_QUOTES SQL mode enabled. What happens?
  • A. Inserts successfully
  • B. Error: unknown column 'Aarav'
  • C. Inserts NULL for name
  • D. Silently ignores the row
Answer: B
B is correct. With ANSI_QUOTES on, double quotes delimit IDENTIFIERS (column/table names), not strings. MySQL would treat "Aarav" as a column name and complain that such a column doesn't exist. Always use single quotes for string literals.
MCQ 18
What is the main reason to prefer a multi-row INSERT over multiple single-row INSERTs?
  • A. Multi-row INSERT uses less disk space
  • B. Multi-row INSERT is a single round-trip to the server and faster
  • C. Multi-row INSERT automatically validates duplicates
  • D. Single-row INSERT is not allowed for tables with more than 3 columns
Answer: B
B is correct. Each individual INSERT is a separate round-trip (client → server → ack). A multi-row INSERT combines them into one request, dramatically reducing network overhead. For 1000 rows this can be 10x-100x faster.
MCQ 19
Which install command works for MySQL on Ubuntu?
  • A. brew install mysql
  • B. apt-get install mysql
  • C. sudo apt install mysql-server
  • D. choco install mysql
Answer: C
C is correct. On Ubuntu/Debian the package is mysql-server, installed via apt with sudo. Option A is Homebrew (Mac). Option D is Chocolatey (Windows). Option B is close but misses sudo and the correct package name.
MCQ 20
You run mysql -u root -p school < seed.sql. What happens?
  • A. Opens the mysql client interactively
  • B. Exports the school database into seed.sql
  • C. Imports and executes the SQL in seed.sql against the school database
  • D. Deletes school.sql after running
Answer: C
C is correct. The < shell redirection pipes seed.sql as stdin to the mysql client, and the school argument sets the default database. Every statement in the file is executed. To export, you would use mysqldump, not mysql.

Challenge 5: Inspect an Unfamiliar Database

Medium
A senior hands you SSH access to a MySQL server and says "there's a database called ecommerce with some tables. Tell me what's in it." Write the sequence of SQL commands you would run to discover: which databases exist, what tables are in ecommerce, what columns each table has, and how many rows are in one specific table called orders.
Sample Input
(No input)
Sample Output
A sequence of discovery commands with expected outputs.
Use exploration commands, not guessing.
-- Step 1: Which databases exist?
SHOW DATABASES;

-- Step 2: Switch to the one we care about.
USE ecommerce;

-- Step 3: What tables are inside?
SHOW TABLES;

-- Step 4: For each table, see the structure.
DESCRIBE orders;
DESCRIBE customers;
DESCRIBE products;

-- Step 5: Count rows in a specific table.
SELECT COUNT(*) AS total_orders FROM orders;

-- Step 6: Peek at the first few rows.
SELECT * FROM orders LIMIT 5;

-- This sequence — SHOW DATABASES, USE, SHOW TABLES, DESCRIBE,
-- COUNT(*), LIMIT 5 — is the universal "orient yourself in a new DB" workflow.
