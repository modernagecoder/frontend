Question 13
Hard
What is the single-file source-code execution feature introduced in Java 11? What are its limitations?
It lets you run a .java file directly with the java command.
Java 11 introduced the ability to run single-file Java programs directly: java HelloWorld.java compiles the source in memory and executes it without producing a .class file on disk. Limitations: (1) Only works for single-file programs. (2) Cannot reference classes in other source files. (3) Cannot use external library JARs. (4) Recompiles every time (no caching). (5) Not suitable for production code.

She needs to set environment variables and open a new terminal.

(3) Edit the Path system variable and add %JAVA_HOME%\bin. (4) Click OK on all dialogs. (5) Open a new Command Prompt (the old one has the old PATH). (6) Run javac -version to verify.

Mixed & Application Questions

Question 1
Easy
What file extension do Java source code files have? What extension do compiled bytecode files have?
One is the code you write, the other is what the compiler produces.
Java source code files have the .java extension. Compiled bytecode files have the .class extension. The compiler (javac) converts .java files into .class files.
Question 2
Easy
Vikram downloads and installs the JRE but cannot compile his Java program. Why?
What does the JRE include vs what does the JDK include?
The JRE (Java Runtime Environment) only includes the JVM and standard libraries needed to run Java programs. It does not include the javac compiler needed to compile Java programs. Vikram needs to install the JDK (Java Development Kit) which includes both the JRE and development tools.
Question 3
Medium
Ananya has two Java installations on her Windows machine: JDK 8 and JDK 21. When she types `java -version`, it shows Java 8. But she wants to use Java 21. How should she fix this?
The system uses whichever Java comes first in the PATH.
Ananya should: (1) Update JAVA_HOME to point to the JDK 21 directory. (2) Ensure %JAVA_HOME%\bin appears before the JDK 8 path in the PATH variable. (3) Alternatively, remove JDK 8 from PATH entirely if she no longer needs it. (4) Open a new terminal and verify with java -version.
Question 4
Medium
Write the terminal commands (not Java code) to: (a) create a directory called 'bin', (b) compile a file called Calculator.java into the bin directory, and (c) run the compiled Calculator class from the bin directory.
Use mkdir, javac with -d, and java with -cp.
mkdir bin
javac -d bin Calculator.java
java -cp bin Calculator
Question 5
Medium
What is the standard directory structure for a Java project? What goes in src/, bin/, and lib/?
Source code, compiled classes, and external libraries each have their directory.
src/ contains Java source code (.java files), organized in package directories. bin/ (or target/classes/) contains compiled bytecode (.class files), mirroring the package structure. lib/ contains external library JAR files that the project depends on.
Question 6
Hard
Explain the difference between compiling with `javac HelloWorld.java` and running with `java HelloWorld.java` (Java 11+). What happens internally in each case?
One produces a .class file on disk, the other compiles in memory.
javac HelloWorld.java compiles the source file and writes HelloWorld.class to disk. You then run it separately with java HelloWorld. java HelloWorld.java (Java 11+) compiles the source in memory, does NOT produce a .class file on disk, and immediately executes the program. The in-memory compilation happens every time you run the command (no caching).
Question 7
Hard
Aarav is setting up IntelliJ IDEA for the first time. He creates a new project but the IDE says 'No JDK found.' What should he do? How does IntelliJ detect the JDK?
IntelliJ needs to know where the JDK is installed. It can auto-detect or you can specify manually.
Aarav should: (1) Click the SDK dropdown in the New Project dialog and select 'Download JDK' (IntelliJ can download it for you). (2) Alternatively, click 'Add JDK' and browse to his JDK installation directory. (3) If JAVA_HOME is set correctly, IntelliJ often detects it automatically. IntelliJ looks for JDKs in standard locations and checks the JAVA_HOME environment variable.
Question 8
Hard
What is a JAR file? How do you run a Java program packaged as a JAR?
JAR stands for Java Archive. It bundles .class files into a single file.
A JAR (Java Archive) file is a ZIP-format archive that bundles compiled .class files, resources, and metadata into a single file for distribution. It is created using the jar command: jar cf MyApp.jar -C bin .. To run an executable JAR: java -jar MyApp.jar. The JAR's manifest file (META-INF/MANIFEST.MF) specifies the main class to execute.
Question 9
Medium
Why should you install the latest LTS version of Java rather than the latest non-LTS version?
Think about support duration and stability.
LTS (Long-Term Support) versions receive security updates and bug fixes for several years, while non-LTS versions only receive 6 months of support. LTS versions are more stable, better documented, widely supported by all tools and libraries, and are what companies use in production. Current LTS versions: Java 11, 17, and 21.
Question 10
Easy
Can you write and compile Java programs without an IDE? What tools do you minimally need?
An IDE is convenient but not required.
Yes, you can write Java programs with any text editor (Notepad, Vim, Nano, Sublime Text) and compile them using the javac command in the terminal. You minimally need: (1) A text editor to write .java files. (2) The JDK installed and in your PATH for javac and java commands.
Question 11
Hard
What is the difference between Homebrew installation (`brew install --cask temurin`) on macOS and manual installation? Are there any advantages to using a package manager?
Package managers automate download, installation, PATH configuration, and updates.
Homebrew automates the entire JDK installation: it downloads the correct binary for your architecture, installs it in the standard location, configures symlinks so java and javac are in PATH, and makes future updates easy (brew upgrade temurin). Manual installation requires downloading the binary, extracting it, setting JAVA_HOME, and updating PATH yourself. The advantage of a package manager is consistency, easy updates, and automated PATH configuration.

Multiple Choice Questions

MCQ 1
Which command compiles a Java source file?
  • A. java
  • B. javac
  • C. javadoc
  • D. jar
Answer: B
B is correct. javac (Java Compiler) compiles .java source files into .class bytecode files. java (A) runs compiled programs. javadoc (C) generates documentation. jar (D) creates archive files.
MCQ 2
What does JAVA_HOME typically point to?
  • A. The JRE installation directory
  • B. The JDK installation directory
  • C. The bin directory inside JDK
  • D. The user's home directory
Answer: B
B is correct. JAVA_HOME should point to the root directory of the JDK installation (e.g., C:\Program Files\Java\jdk-21). It should NOT point to the JRE (A), the bin directory (C), or the user's home (D). Build tools use JAVA_HOME to locate the JDK.
MCQ 3
What file is produced when you run `javac HelloWorld.java`?
  • A. HelloWorld.exe
  • B. HelloWorld.class
  • C. HelloWorld.jar
  • D. HelloWorld.byte
Answer: B
B is correct. The Java compiler produces a .class file containing bytecode. Java does not produce .exe files (A) or .byte files (D). A .jar file (C) is an archive created separately using the jar tool.
MCQ 4
Which of the following is a free, open-source Java IDE?
  • A. IntelliJ IDEA Ultimate
  • B. Eclipse
  • C. Xcode
  • D. PyCharm Professional
Answer: B
B is correct. Eclipse is completely free and open source. IntelliJ IDEA Ultimate (A) requires a paid license (though Community Edition is free). Xcode (C) is for Apple/iOS development. PyCharm Professional (D) is for Python and requires a license.
MCQ 5
What is the correct command to run a compiled Java class named Main?
  • A. java Main.class
  • B. java Main.java
  • C. java Main
  • D. javac Main
Answer: C
C is correct. The correct command is java Main (class name without extension). java Main.class (A) would look for a class literally named "Main.class". java Main.java (B) would try to compile and run from source (Java 11+). javac Main (D) is the compiler, not the runner.
MCQ 6
If your file is named MyProgram.java, what MUST the public class inside be named?
  • A. myprogram
  • B. MyProgram
  • C. Myprogram
  • D. Any name is fine
Answer: B
B is correct. The public class name must exactly match the file name, including capitalization. MyProgram.java must contain public class MyProgram. Options A and C have different capitalization, which will cause a compiler error. Option D is wrong because Java enforces this naming rule.
MCQ 7
What does the `-d bin` flag do in `javac -d bin HelloWorld.java`?
  • A. Debugs the program
  • B. Deletes the bin directory
  • C. Places the compiled .class file in the bin directory
  • D. Downloads dependencies to bin
Answer: C
C is correct. The -d flag specifies the destination directory for compiled .class files. -d bin tells the compiler to place output files in the bin/ directory. It does not debug (A), delete (B), or download (D).
MCQ 8
What does `-cp` stand for in the java command?
  • A. Copy
  • B. Compile path
  • C. Classpath
  • D. Current path
Answer: C
C is correct. -cp is shorthand for -classpath. It specifies the directories and JAR files where the JVM should look for compiled .class files. It is not about copying (A), compilation (B), or the current directory (D).
MCQ 9
Which Java version introduced single-file source-code execution (`java File.java`)?
  • A. Java 8
  • B. Java 9
  • C. Java 11
  • D. Java 17
Answer: C
C is correct. Java 11 (JEP 330) introduced the ability to run single-file Java programs directly with java File.java. Java 8 (A) and 9 (B) require explicit compilation. Java 17 (D) is an LTS release but this feature came in Java 11.
MCQ 10
Which JDK distribution is provided by the Eclipse Foundation?
  • A. Amazon Corretto
  • B. Eclipse Temurin
  • C. Azul Zulu
  • D. Microsoft Build of OpenJDK
Answer: B
B is correct. Eclipse Temurin (formerly AdoptOpenJDK) is the Eclipse Foundation's distribution of OpenJDK via the Adoptium project. Amazon Corretto (A) is from Amazon. Azul Zulu (C) is from Azul Systems. Microsoft Build (D) is from Microsoft.
MCQ 11
What error do you get if you compile with JDK 21 but try to run with JRE 8?
  • A. ClassNotFoundException
  • B. UnsupportedClassVersionError
  • C. NoSuchMethodError
  • D. FileNotFoundException
Answer: B
B is correct. UnsupportedClassVersionError occurs when bytecode compiled for a newer JDK version is run on an older JRE. The JRE cannot interpret bytecode from a version higher than its own. ClassNotFoundException (A) means the class file was not found. NoSuchMethodError (C) means a method is missing. FileNotFoundException (D) is for file I/O operations.
MCQ 12
On Windows, after setting JAVA_HOME and PATH, the `javac` command still does not work in the current terminal. What is the most likely cause?
  • A. The JDK is not installed
  • B. The existing terminal has the old PATH; open a new one
  • C. You need to restart the computer
  • D. JAVA_HOME is read-only
Answer: B
B is correct. On Windows, existing terminal windows retain the PATH values from when they were opened. After changing environment variables, you must open a new Command Prompt or PowerShell window. Restarting the computer (C) would also work but is unnecessary. The JDK being not installed (A) is possible but less likely if you just set JAVA_HOME.
MCQ 13
What is the purpose of the META-INF/MANIFEST.MF file inside a JAR?
  • A. It contains the source code
  • B. It lists metadata like the main class and version information
  • C. It contains compilation errors
  • D. It is the bytecode for the main class
Answer: B
B is correct. The manifest file contains metadata about the JAR, including the main class to execute (Main-Class attribute), the classpath, version information, and other properties. It does not contain source code (A), errors (C), or bytecode (D).
MCQ 14
Which of the following is NOT a valid way to install JDK on Linux?
  • A. sudo apt install openjdk-21-jdk
  • B. Download .tar.gz from Oracle and extract manually
  • C. sdk install java 21-tem (using SDKMAN)
  • D. pip install java-jdk-21
Answer: D
D is correct (this is NOT valid). pip is the Python package manager and cannot install the Java JDK. Options A (apt), B (manual download), and C (SDKMAN) are all valid methods to install JDK on Linux.
MCQ 15
When you run `javac src/*.java -d bin`, what happens?
  • A. Only the first .java file is compiled
  • B. All .java files in src/ are compiled and .class files are placed in bin/
  • C. An error occurs because wildcards are not supported
  • D. The bin directory is deleted and recreated
Answer: B
B is correct. The wildcard *.java matches all Java source files in the src/ directory. The compiler processes all of them, resolves dependencies between classes, and places the compiled .class files in the bin/ directory. Wildcards are supported by both javac and the shell.
MCQ 16
Which of the following is the recommended JDK version type for production and learning?
  • A. Beta release
  • B. Early Access
  • C. Long-Term Support (LTS)
  • D. Non-LTS feature release
Answer: C
C is correct. LTS versions receive extended support and security patches for years. Beta (A) and Early Access (B) versions are for testing unreleased features. Non-LTS (D) versions only get 6 months of support. Always use LTS for stability.
MCQ 17
Which command checks where the java executable is located on Windows?
  • A. which java
  • B. where java
  • C. find java
  • D. locate java
Answer: B
B is correct. On Windows, where java shows the full path of the java executable. On Linux/Mac, which java (A) is the equivalent command. find (C) searches for files. locate (D) uses a file database on Linux.
MCQ 18
What is SDKMAN and why is it useful for Java developers?
  • A. A Java framework for building web apps
  • B. A tool for managing multiple JDK versions on one machine
  • C. An IDE plugin for code generation
  • D. A Java profiling and debugging tool
Answer: B
B is correct. SDKMAN (Software Development Kit Manager) allows developers to install, switch between, and manage multiple JDK versions and other JVM-based tools (Maven, Gradle, Kotlin). Commands like sdk install java 21-tem and sdk use java 17-tem make version management effortless. It is not a framework (A), IDE plugin (C), or profiler (D).
